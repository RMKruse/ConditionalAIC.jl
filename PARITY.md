# PARITY.md — `cAIC4` Parity Matrix

The authoritative scope definition (CLAUDE.md §10). Every public function and
supported model family of `cAIC4`, mapped to a `cAIC.jl` milestone and a status.
**"Feature parity" means every in-scope row is 🟢.** Divergences are not tracked
here — they live in `DECISIONS.md`; architectural choices live in `docs/adr/`.

Terms (Scoring / Comparison / Search, steinian / bootstrap, singular / reduced)
are defined in [CONTEXT.md](CONTEXT.md). Milestones M1–M6 are defined in
CLAUDE.md §11.

## Status legend

| Symbol | Meaning |
|--------|---------|
| 🟢 | done — implemented **and** validated against the R reference |
| 🟡 | in progress / partial |
| 🟦 | planned — scope **resolved** (design grilled, not yet built) |
| ⬜ | planned — scope **not yet grilled** |
| 🚫 | deliberately out of scope / deferred (see note) |

## Near-term scope

The committed near-term target is **LMM scoring and its immediate selection
layer** — i.e. the M2 → M2.5 → M4 spine on Gaussian `LinearMixedModel`s. GLMM
(M3), model averaging (M4.5), and additive models (M5) are deliberately deferred
to their own milestones; M5 is decided late by design (CLAUDE.md §11). The rows
below reflect that: the LMM spine is 🟦 (resolved), GLMM/additive rows are ⬜/🚫.

| `stepcAIC(...)` | `stepcaic(m, data; …)` | M4 | 🟦 | **Search**: random-effects structure **only** — the fixed-effects part is held constant on every candidate, matching `cAIC4`'s `(g)lmer` path (its `fixEfCandidates` are `gamm4`-smooth-only → M5). Full non-`gamm4` option surface, **serial** (parallelism deferred). | `modelAvg` / `predictMA` / `summaryMA` (+ internal `getWeights` / `weightOptim`) | `modelavg` / `predictma` / `getweights` (`summaryMA` folded into `ModelAvgResult` `show`; `weightoptim` internal) | M4.5 | 🟢 | **Averaging**: cAIC-weighted model combination. Its own milestone M4.5 (CLAUDE.md §11, amended 2026-05-27). **Resolved design (grilled 2026-05-31):** Gaussian **LMM only** (the Zhang/Mallows weight objective `(y−μw)ᵀ(y−μw)+2σ²(w·ρ)` is Gaussian by construction; no R ground truth for GLMM averaging → deferred, like FE *Search*). Public surface `modelavg`/`predictma`/`getweights` (the `summaryMA` report is the `ModelAvgResult` display, not a function); result structs `ModelAvgResult{T}` (name-keyed model-averaged `fixeff`/`raneff`, weights, candidate cAICs, opt-mode) and `WeightResult{T}` (weights, objective value, runtime `duration`). Level-2-validated end-to-end vs `cAIC4` `modelAvg(opt=FALSE)` on the sleepstudy corr/uncorr/int set (`modelavg_level2.h5`; atol=1e-3; the `digits=2` weight-rounding divergence absorbed by the band, DECISIONS 2026-05-31). **Zhang-optimal path landed (#50):** `getweights(res::ModelAvgResult)` → `WeightResult{T}` and `modelavg(…; weights=:zhang)` both implemented. `_getweights_raw`/`_weightoptim` are faithful Julia transcriptions of `cAIC4`'s `getWeights`/`.weightOptim` (ADR-0007). Level-1-validated: synthetic `(y, mu, rho, sigma_sq)` fed identically to both R and Julia; weight vectors and objective agree `rtol=1e-6, atol=1e-10` on two well-conditioned test cases (DECISIONS 2026-05-31). `@inferred` type-stable. Full-precision df vs `cAIC4`'s `digits=2` rounding (documented divergence, DECISIONS 2026-05-31). **Default wired + Level-2 anchor landed (#51):** `modelavg` default scheme is now `:zhang` (mirroring `cAIC4`'s `opt=TRUE` default). Level-2 end-to-end anchor vs `cAIC4::modelAvg(opt=TRUE)` on the well-conditioned sleepstudy (full-slope vs intercept-only, genuinely different conditional means, `MᵀM ≻ 0`): weight vector `atol=1e-2`, objective `rtol=1e-4` (`zhang_modelavg_level2.h5`; DECISIONS 2026-05-31). `getweights` fast path: `:zhang` result caches `WeightResult` in `ModelAvgResult.weightresult` — no re-scoring on re-call. **`predictma` landed (#52):** `predictma(res, newdata; new_re_levels=:error)` → `Vector{T}`, the weighted conditional prediction `ŷ^MA = Σ wᵢ predict(mᵢ, D*)` (port of `cAIC4`'s `predictMA`). The `:error` default mirrors `lme4`'s `allow.new.levels=FALSE` (overriding `MixedModels`' `:missing` default; `:population`/`:missing` opt-in), a recorded divergence (DECISIONS 2026-05-31). `@inferred` type-stable. **Level-2 stable-functional anchor** vs `cAIC4::modelAvg(opt=TRUE)`+`predictMA` on a well-conditioned set (prediction **and** weights) **and** a nested Orthodont-style set (prediction only — the §7 stable-functional discipline): per-observation `atol=5e-3`, measured worst-case `|Δŷ^MA|=1.63e-3` from the lme4↔MM fit discrepancy (`predictma_level2.h5`; DECISIONS 2026-05-31). **`summaryMA` report folded into `show` (#53, revised):** the port of `cAIC4`'s `summaryMA` is the `text/plain` `Base.show` of `ModelAvgResult` — **there is no standalone `summaryma` function** (a deliberate parity divergence: the named function is gone, the functionality is reached idiomatically by displaying the result; DECISIONS 2026-05-31). Displaying a result prints the candidate model formulas, a per-candidate **cAIC + weight** table (weights `round(·; digits=6)`, matching `summaryMA`; the cAIC column extends `summaryMA`'s weights-only listing), the model-averaged fixed effects, and — **always** — the averaged random effects keyed `grouping[level] term`. Pure display (no numerics beyond the already-validated `fixeff`/`weights`/`caics`/`raneff`), so no R fixture. Three recorded display divergences (DECISIONS 2026-05-31): candidate **formulas** stand in for `summaryMA`'s `z$call` (not retained by `ModelAvgResult`); the random-effects heading is corrected from `summaryMA`'s copy-pasted "…Fixed Effects" label (an upstream bug, not transcribed — ADR-0007 decision 3); and there is no `randeff` toggle (a `show` method takes no kwarg — the full report is the display; raw `fixeff`/`raneff`/`weights` remain fields). **M4.5 complete.** |

## df estimators & B-source

| `cAIC4` | `cAIC.jl` | Status | Notes |
|---------|-----------|--------|-------|
| `method=NULL` (auto by family) | `method=:auto` | 🟢 | Gaussian: resolves to `:steinian` (validated #8). GLMM: dispatches to family-specific estimator (Poisson → Chen-Stein, Bernoulli → Efron; #31). Non-Poisson/Bernoulli families require `method=:bootstrap`. |
| steinian | `method=:steinian` | 🟢 (Gaussian) / not-a-kwarg (GLMM) | For Gaussian: `:steinian` is the explicit kwarg for the analytic GK correction. For GLMM, `:steinian` is not a valid kwarg — family-specific estimators are selected via `:auto` (no `:steinian` override exists in M3 scope; `cAIC4`'s "steinian" for GLMM is the auto-dispatch, not a separate option). |
| `conditionalBootstrap` | `method=:bootstrap` | 🟢 | Gaussian: parametric bootstrap (#12). GLMM: conditional bootstrap (#30, #31); validated vs. cAIC4 `conditionalBootstrap` (atol=2.0). Reachable end-to-end through `caic` for Poisson, Bernoulli, and multi-trial Binomial (binomial `ℓ_cond` via `condloglik_binomial`). `rng` kwarg for reproducibility. |
| `analytic=TRUE` (closed-form B) | `hessian=:analytic` | 🟢 | Default B-source; no derivative dependency. Level-2-validated (#8). |
| `analytic=FALSE` (lifted lme4 Hessian) | `hessian=:forwarddiff` / `:finitediff` | 🟢 | No lme4 Hessian to lift in MM; B computed at cAIC-time (ADR-0002, #11). `:finitediff` self-drives FiniteDiff over MM's stable profiled objective and reproduces `analytic=FALSE` ρ to FD accuracy, Level-2-validated (atol=1e-3, DECISIONS); `:forwarddiff` rides the experimental ext, diverging by σ-freezing. No bit-match to `analytic=FALSE` (DECISIONS + ADR-0001). |
| `sigma.penalty` | `sigmapenalty::Int` | 🟢 | Carried through unchanged; verified to shift ρ by one per unit (#8). |
